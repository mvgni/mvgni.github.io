Proud and Torn

Proud and Torn is a visual novel detailing Hungarian history from the point of view of a peasant family in the Hungarian countryside. The project contains hundreds of photographs that were altered, isolated into parts, put into animation, or crafted into photomontages. The project is designed to resemble a personalized scrapbook but utilizes groundbreaking web design tactics.
My role in the project was mainly comprised of optimizing each page for mobile devices, but I also had tasks that included making pages ADA compliant, converting SVG images to selectable text, and transferring inline styles to external SCSS stylesheets.

You can view this project at proudandtorn.org

Leaflet Watershed Application

This is a web application used to visualize watershed catchment boundaries and export user-specified options related to each catchment area as a CSV file.

There are two views for this applicaion: the Map view and the Text view. The Map view allows the user to interact with a map to visualize individual catchment boundaries by clicking, or selecting areas by drawing a rectangle or polygon. The boundaries are generated using data from a custom Web API.
The Text view allows for much larger catchment queries such as counties, states, and regions. Both views retrieve certain data from the custom API based on the selected catchments, using two multiselect boxes. The application generates a CSV file with all the data, which can be manipulated and loaded back in through the Text view.

Gunnar Whisler - portfolio

This is a portfolio website that I designed and created for my partner using HTML5, Sass, jQuery, and Webpack.

This website is meant to showcase his accomplishments throughout his experience in the Higher Education/Student Affairs Masters program at Oregon State University. He ended up submitting and defending this portfolio as his capstone for acquiring his M.Ed.

The site is extremely text-heavy, so I wanted to design the site so it would not only look visually appealing and minimal to keep the reader focused on the text, but also to make the text and line height large enough so the reader wouldn't have to struggle to read anything.

This site was also built with a mobile-first ideology, so even though most of the people looking at his portfolio would be doing so from a computer, it will still function correctly on different screen sizes just in case.

Live Site
The text on most of the pages was very long, so I broke it up into sections and added static anchor links in the form of dots on the right side of the screen, which reveal the section titles on hover.
